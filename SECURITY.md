# Security Policy

## Supported Versions

| Version | Supported |
| ------- | --------- |
| Latest  | Yes       |

## Reporting a Vulnerability

If you discover a security vulnerability in Roost, please report it responsibly.

Do not open a public issue for security vulnerabilities. Use the Roost repository's private vulnerability reporting channel when available. If that is not available, contact the maintainers privately through the contact path listed for the project.

### What to include

- A description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix, if any

### What to expect

- Maintainers aim to acknowledge reports promptly
- Confirmed issues get a fix plan based on severity and maintainer availability
- Credit in the release notes unless you prefer to remain anonymous

## Scope

This policy applies to the Roost macOS application, its bundled scripts, its local remote-server features, and its build/release tooling. For vulnerabilities in [libghostty](https://github.com/ghostty-org/ghostty), report them to the Ghostty project directly.
